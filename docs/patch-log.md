| 2026-04-23 | **Toggle Mode Lab/One cassé — `MODE_TOGGLE_COOKIE = undefined` côté RSC** : Le cookie `mpp_active_view` était bien posé (visible DevTools + middleware) mais le layout serveur retournait toujours `activeMode='lab'`. Cause racine : `MODE_TOGGLE_COOKIE` était ré-exporté depuis `mode-toggle.tsx` (`'use client'`) via `packages/ui/src/index.ts`. Next.js 15 transforme les constantes de fichiers client en références client → côté RSC la valeur est `undefined`. Donc le layout faisait `cookieStore.get(undefined)` → `undefined` toujours. Fix : déplacer `MODE_TOGGLE_COOKIE` dans `mode-toggle-constants.ts` (sans directive), exporter depuis là dans `index.ts`. Bonus : création `mode-toggle-action.ts` avec `'use server'` pour poser le cookie côté serveur + `window.location.replace('/')` côté client pour full reload. Également corrigé : turbo.json `"ui": "tui"` → `"stream"` (TUI crashait sur PowerShell), scripts `dev:hub`/`dev:client`, `'use client'` ajouté à `use-mobile.ts`, `use-system-health.ts`. | `packages/ui/src/components/{mode-toggle.tsx, mode-toggle-action.ts, mode-toggle-constants.ts, mode-toggle.test.tsx, use-mobile.ts}`, `packages/ui/src/index.ts`, `packages/modules/admin/hooks/use-system-health.ts`, `turbo.json`, `package.json` | RSC-003 : constantes depuis `'use client'` = undefined au RSC (piège silencieux majeur) | Oui → lessons-learned |
| 2026-04-20 | **get-clients crash clients disparus** : `.parse()` sur `ClientListItemSchema` lève ZodError quand un client a un champ invalide. Fix : remplacé par `.safeParse()` en boucle `for...of`, les clients invalides sont skippés+loggués, les valides sont retournés. Commit `fd8d551`. | `packages/modules/crm/actions/get-clients.ts` | ZodError non-sérialisable en RSC (leçon RSC-001 confirmée) | Oui → lessons-learned |
